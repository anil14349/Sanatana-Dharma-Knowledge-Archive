## 1. Editorial Philosophy

### 1.1 Core Editorial Mandate

The Editorial System exists to serve a single purpose:

**Ensure that every knowledge object published in SDKA meets the highest standards of accuracy, clarity, consistency, and respect.**

This mandate is supported by a hierarchy of values:

1. **Accuracy First** — Claims must be verified against sources
2. **Clarity Second** — Complex ideas must be made understandable
3. **Consistency Third** — Tone and structure must be uniform
4. **Style Fourth** — Writing should be polished and professional
5. **Engagement Last** — Content should be accessible but never sacrifices above values

### 1.2 Editorial Principles

**Respect for Sources**
- Edit to clarify, not to change meaning
- Preserve author voice while improving clarity
- Maintain all citations and source attributions
- Verify quotes and references

**Intellectual Honesty**
- Flag uncertainty and ambiguity
- Preserve multiple viewpoints
- Ensure nuance in controversial topics
- Distinguish fact from interpretation

**Accessibility**
- Simplified complex concepts
- Defined specialized terms
- Improved organization without losing rigor
- Enhanced readability without dumbing down

**Cultural Sensitivity**
- Respectful treatment of traditions
- Appropriate language for sacred topics
- No condescension or ridicule
- Accurate cultural representation

**Continuous Improvement**
- Feedback loops for improvement
- Regular quality audits
- Standard evolution as project matures
- Learning from community feedback

## 2. Editorial Workflow

### 2.1 Submission Process

**Step 1: Research Completion**
- Researcher completes knowledge object
- All sections filled (per Knowledge Object Specification)
- All citations verified
- Researcher confidence level noted

**Step 2: Self-Review**
- Researcher uses quality checklist
- Verifies all standards met
- Marks status as "Ready for Review"

**Step 3: Editorial Submission**
- File submitted to editorial queue
- Metadata verified (all fields complete)
- Basic formatting checked
- Assigned to editor

### 2.2 Editorial Review Stages

#### Stage 1: Intake Review (1-2 days)

**Purpose:** Verify submission is complete and identifiable issues

**Reviewer checks:**
- [ ] All required sections present
- [ ] Metadata complete and accurate
- [ ] Basic formatting correct
- [ ] File naming follows convention
- [ ] All obvious errors noted

**Outcomes:**
- **Approve** → Proceed to full review
- **Request revisions** → Return to researcher for corrections
- **Reject** → Request substantial rework before resubmission

#### Stage 2: Accuracy Review (3-5 days)

**Purpose:** Verify all factual claims against sources

**Reviewer process:**
1. **Source Verification**
   - Check every citation
   - Verify quotes are accurate
   - Confirm page numbers correct
   - Validate translation versions

2. **Factual Accuracy**
   - Verify spellings and dates
   - Check geographic information
   - Confirm cultural facts
   - Verify technical information

3. **Scope Appropriateness**
   - Ensure claims are sourced
   - Identify unsupported assertions
   - Flag speculation not labeled as such
   - Check for missing context

**Reviewer outputs:**
- Detailed comments on accuracy issues
- List of required corrections
- Queries for researcher
- Strength of evidence assessment

**Researcher response time:** 3-5 days

#### Stage 3: Consistency & Clarity Review (3-5 days)

**Purpose:** Ensure clarity and consistency with archive standards

**Reviewer checks:**

**Consistency:**
- [ ] Tone matches archive voice
- [ ] Definitions align with other entries
- [ ] Cross-references are accurate
- [ ] Formatting follows standards
- [ ] Metadata is consistent with content

**Clarity:**
- [ ] Specialized terms are defined
- [ ] Complex ideas explained simply
- [ ] Examples enhance understanding
- [ ] Organization is logical
- [ ] Transitions flow smoothly

**Respect & Appropriateness:**
- [ ] Respectful treatment of traditions
- [ ] Neutral on disputed matters
- [ ] Appropriate language for sacred topics
- [ ] No problematic assumptions
- [ ] Inclusive and equitable perspective

**Reviewer outputs:**
- Suggestions for clarity improvements
- Structural/organizational notes
- Tone adjustments if needed
- Acknowledgment of strengths

**Researcher response time:** 3-5 days

#### Stage 4: Final Review (2-3 days)

**Purpose:** Final verification and approval

**Final reviewer:**
- [ ] Verifies all corrections incorporated
- [ ] Checks resolved issues
- [ ] Spot-checks accuracy
- [ ] Confirms quality standards met

**Approval decision:**
- **Approved** → Content ready for publication
- **Conditionally approved** → Minor issues to address before archive
- **Return for revision** → Significant issues need addressing

### 2.3 Complete Workflow Timeline

```
Submission → Intake (1-2 days) → Accuracy Review (3-5 days) 
→ Researcher Revisions (3-5 days) → Clarity Review (3-5 days)
→ Researcher Revisions (3-5 days) → Final Review (2-3 days)
→ APPROVED → Archive Storage

Total typical timeline: 20-30 days
Expedited (if urgent): 10-15 days
```

## 3. Review Standards

### 3.1 Accuracy Standards

**What must be verified:**

| Claim Type | Verification Method | Confidence Level Required |
|-----------|-------------------|--------------------------|
| Direct quote | Original source | High (100%) |
| Specific fact (date, number, name) | Primary source | High (100%) |
| General statement | Multiple sources | Medium (75%+) |
| Tradition holds | Scholarly consensus | Medium (75%+) |
| Scholar argues | Academic source | Medium (75%+) |
| Regional variation | Primary evidence | Medium (75%+) |
| Uncertain information | Clearly labeled | Acknowledged in text |

**Red flags that require correction:**

❌ Claims without sources  
❌ Quotes taken out of context  
❌ Misattributed sources  
❌ Conflicting information not addressed  
❌ Presented as fact when uncertain  
❌ Missing crucial context  
❌ Over-generalization from limited evidence  

### 3.2 Clarity Standards

**Clear writing has:**

✅ Single main idea per section  
✅ Specialized terms defined  
✅ Complex ideas broken down  
✅ Examples provided for difficult concepts  
✅ Logical progression from simple to complex  
✅ Clear transitions between ideas  
✅ Active voice (mostly)  
✅ Concrete language over abstract  

**Unclear writing has:**

❌ Multiple competing ideas in one section  
❌ Undefined technical terms  
❌ Overly complex sentences  
❌ No examples or only abstract examples  
❌ Logical leaps (reader can't follow)  
❌ Abrupt transitions  
❌ Passive voice throughout  
❌ Overly abstract language  

### 3.3 Tone Standards

**Appropriate tone for SDKA:**

**Use:**
- Scholarly but accessible
- Respectful and reverent (for sacred topics)
- Authoritative but humble about uncertainties
- Curious and engaging
- Never condescending

**Avoid:**
- Overly technical jargon
- Dismissive or derisive language
- False certainty
- Preaching or proselytizing
- Overly casual or colloquial
- "I" or "we" voice (use passive or third person)

### 3.4 Structure Standards

**Knowledge objects should:**

✅ Have clear hierarchy (headings used correctly)  
✅ Begin with overview/summary  
✅ Progress logically  
✅ Use sections to organize complex content  
✅ Include citations and sources  
✅ Have cross-references to related topics  
✅ Acknowledge limitations and gaps  
✅ End with closure or transition to related topics

## 4. Editorial Roles

### 4.1 Editorial Team Structure

**Senior Editor**
- Oversees editorial standards
- Handles complex or controversial topics
- Resolves disputes between editors
- Manages editorial policy
- Trains new editors

**Subject Matter Editors**
- Focused expertise in specific domains
- Verify accuracy in their domain
- Understand nuances and complexities
- Mentor researchers in their area
- Examples: Vedic specialist, Philosophy editor, Temple expert

**General Editors**
- Handle clarity and consistency review
- Manage workflow
- Handle routine corrections
- Support senior editors
- Check style and formatting

**Copy Editors**
- Final pass for grammar and style
- Check formatting consistency
- Verify citations format
- Ensure consistency with guidelines

### 4.2 Editor Qualifications

**All editors must have:**
- [ ] Deep knowledge of Sanātana Dharma
- [ ] Understanding of SDKA mission and values
- [ ] Strong writing and editing skills
- [ ] Ability to research and verify information
- [ ] Cultural sensitivity and respect
- [ ] Commitment to accuracy and clarity
- [ ] Attention to detail

**Subject matter editors additionally:**
- [ ] Advanced expertise in specific domain
- [ ] Published works or recognized scholarship (preferred)
- [ ] Network within scholarly community
- [ ] Ability to mentor researchers

**Senior editors additionally:**
- [ ] 2+ years editorial experience
- [ ] Demonstrated leadership
- [ ] Understanding of editorial policy
- [ ] Ability to handle conflicts

### 4.3 Editor Responsibilities

**During Review:**
- Read carefully and completely
- Note all issues clearly
- Provide constructive feedback
- Ask clarifying questions
- Acknowledge strengths
- Explain requested changes
- Remain professional and respectful

**Feedback Quality:**
- Specific examples for corrections
- Tone that encourages improvement
- Balance criticism with acknowledgment
- Clear priority (critical vs. nice-to-have)
- Actionable suggestions

## 5. Quality Assurance Processes

### 5.1 Quality Metrics

Track these metrics for continuous improvement:

**Process Metrics:**
- Average review time per stage
- Number of revisions requested
- Percentage approved first submission
- Percentage requiring multiple revision rounds

**Quality Metrics:**
- Accuracy issues found post-publication
- User corrections submitted
- Citation verification rate
- Sources cited per object

**Consistency Metrics:**
- Tone consistency across archive
- Formatting compliance rate
- Citation style compliance
- Cross-reference accuracy

### 5.2 Periodic Audits

**Monthly audit:**
- Sample 10% of recently published objects
- Spot-check accuracy
- Verify citations
- Check consistency
- Identify patterns needing attention

**Quarterly audit:**
- Comprehensive review of specific topic area
- Systematic accuracy verification
- Citation completeness check
- Identify outdated information
- Assess overall quality trends

**Annual audit:**
- System-wide quality assessment
- Identify improvement opportunities
- Update standards if needed
- Archive retrospective accuracy
- Planning for coming year

### 5.3 Error Tracking

**When errors are found:**

1. **Log the error**
   - What type (accuracy, clarity, citation)
   - Severity (critical, major, minor)
   - Which stage missed it
   - Root cause

2. **Correct the error**
   - Implement fix
   - Verify correction
   - Update version history
   - If published, issue correction notice

3. **Learn from error**
   - Update standards if needed
   - Retrain editors if pattern
   - Improve processes to prevent recurrence

## 6. Style & Consistency Guidelines

### 6.1 Tone Guidelines

**Register:** Scholarly-accessible (academic tone with clear writing)

**Perspective:** Third person (avoid "I" and "we")

**Voice:** Mostly active voice

**Formality:** Professional but not stiff

**Examples:**

✅ GOOD: "The Vedas describe Indra as a warrior deity."  
❌ BAD: "I think the Vedas show Indra as tough."  
❌ BAD: "We find that Indra is described as a warrior."  

✅ GOOD: "Tradition holds that Ganesha removes obstacles."  
❌ BAD: "Obviously, Ganesha gets rid of problems."  

### 6.2 Terminology Standards

**First mention of foreign terms:**
- Provide translation
- Use italics: *Bhagavad Gita*
- Include transliteration in parentheses if helpful: *Bhagavad Gita* (Bhagavadgītā)

**Sacred terms:**
- Show respect through language
- Capitalize properly (Vedas, Puranas, etc.)
- Use correct transliteration
- Explain significance if not obvious

**Specialized terms:**
- Define in first use
- Use consistently
- Cross-reference glossary if complex

### 6.3 Citation Style

**For direct quotes:**
```
"Direct quote here" (Source Name, Section, Translation note if applicable)
```

**For paraphrases:**
```
According to [Source Name], [paraphrased idea]. (Source Name, Location)
```

**In-text citations:**
```
This concept appears in the Vedas (Rigveda, Mandala X, Sukta 90).
```

**Full bibliography format:**
```
## Sources

1. Primary Sources
   - Rigveda, Mandala 10, Sukta 90 (Purusha Sukta)
   - Translation: [specific translation used]

2. Secondary Sources
   - Author Name, *Book Title* (Publisher, Year)
   - Accessed: [date if online source]
```

### 6.4 Formatting Standards

**Headings:**
- Level 1 (#) - Article title
- Level 2 (##) - Major sections
- Level 3 (###) - Subsections
- Level 4 (####) - Sub-subsections (rarely)

**Emphasis:**
- **Bold** - Key terms, definitions, important concepts
- *Italics* - Foreign words, Sanskrit, emphasis
- `Code` - Technical terms, transliterations (use sparingly)

**Lists:**
- Bullet lists for collections
- Numbered for sequences or steps
- Indentation for hierarchy

**Tables:**
- For comparisons and structured data
- Clear headers
- Consistent alignment

## 7. Managing Corrections & Revisions

### 7.1 Post-Publication Corrections

**When errors are found after publication:**

1. **Assess severity**
   - Critical (changes meaning or accuracy)
   - Major (significant error but context clear)
   - Minor (typo or small factual error)

2. **Implement correction**
   - Update knowledge object
   - Update version number
   - Note correction in metadata
   - Update publication date

3. **Communicate correction**
   - If public, issue correction notice
   - If critical, notify subscribers
   - Explain what was corrected
   - Thank person who reported error

### 7.2 Version Control

**Metadata tracking:**
```
---
title: [Topic]
version: 1.2
date_updated: 2026-06-30
change_log:
  - 1.2: Corrected Vedic citation (2026-06-30)
  - 1.1: Added regional variations (2026-06-15)
  - 1.0: Initial publication (2026-06-01)
---
```

**Major versions (1.0, 2.0):**
- Substantial content additions
- Significant reorganization
- Fundamental corrections

**Minor versions (1.1, 1.2):**
- Clarifications
- Small additions
- Error corrections
- Enhanced citations

### 7.3 Revision Requests

**Researcher receives revision request:**
- Clear explanation of needed changes
- Priority/severity indicated
- Deadline for revision
- Editor remains available for questions

**Researcher revises:**
- Address all points
- Explain any disagreements with suggestions
- Resubmit with change notes
- Request clarification if needed

**Editor verifies revisions:**
- Check all points addressed
- Verify quality of revisions
- Move forward if satisfactory
- Request additional revision if needed

Through careful editorial processes and continuous quality improvement, SDKA maintains the highest standards of knowledge presentation while remaining accessible and engaging to diverse audiences.
