# Purpose

This document defines the official SDKA workflow for transforming validated research into public-facing educational content.

**Principle:** Content is always derived from a validated Knowledge Object—not directly from research notes, social media trends, or memory.

---

# Core Workflow

```text
Idea
  ↓
Research Notes
  ↓
Evidence Validation
  ↓
Knowledge Package
  ↓
Evidence Audit
  ↓
R3 – Repository Ready
  ↓
Content Strategy
  ↓
Instagram / Reel / YouTube / Website / Book
  ↓
Publication Review
  ↓
Archive & Analytics
```

---

# Stage 1 – Research Notes

## Inputs
- Research topic
- Primary sources
- Secondary scholarship
- Living traditions
- Field observations (if available)

## Outputs
- Research notes
- Open questions
- Initial bibliography

No public content should be created at this stage.

---

# Stage 2 – Knowledge Package

Convert research into a structured Knowledge Object.

Minimum outputs:

- README.md
- metadata.yaml
- evidence-matrix.md
- bibliography.md
- knowledge-graph.md

Supporting files are added as appropriate.

---

# Stage 3 – Evidence Audit

Before publication:

- Evidence Audit Checklist completed.
- Confidence assigned.
- Misconceptions documented.
- Regional variation recorded.
- Unsupported claims removed or labelled.

Only after passing the audit may the Knowledge Object be marked **R3 – Repository Ready**.

---

# Stage 4 – Content Strategy

Select the best publishing format.

| Goal | Recommended Format |
|------|--------------------|
| Teach a concept | Carousel |
| Correct a myth | Carousel + Reel |
| Explain a ritual | Temple Guidebook |
| Tell a story | Storytelling Carousel |
| Compare traditions | Illustrated Encyclopedia |
| Deep dive | Website / YouTube |

---

# Stage 5 – Content Derivation Rules

Every content asset must reference the Knowledge Object.

For each slide or section, record:

- Supporting Knowledge Object section
- Evidence level
- Confidence
- Visual objective

Example:

| Slide | KO Source | Evidence |
|------|-----------|----------|
| Hook | README | A |
| Soma Sutra | architecture.md | A |
| Chandikeshvara | ritual.md | A/B |
| Myth | misconceptions.md | D |

---

# Instagram Production Standard

Each production brief should include:

- Objective
- Audience
- Page archetype
- Story arc
- Slide-by-slide copy
- Visual prompts
- Caption
- CTA
- Hashtags
- Evidence mapping

---

# Multi-Platform Adaptation

A single Knowledge Object may generate:

- Instagram carousel
- Reel narration
- YouTube script
- Website article
- Podcast outline
- Newsletter
- Future book chapter

All versions must preserve the validated evidence.

---

# Publication Checklist

- [ ] Derived from an R3 Knowledge Object
- [ ] Evidence verified
- [ ] Regional variation acknowledged
- [ ] Misconceptions avoided
- [ ] Caption reviewed
- [ ] Visual prompts completed
- [ ] Internal links updated

---

# Analytics & Feedback

After publication, record:

- Views
- Saves
- Shares
- Comments
- Questions from readers

Useful feedback should become:

- Repository issues
- Open research questions
- Future Knowledge Object updates

---

# Guiding Principles

1. Research before storytelling.
2. Curiosity without sensationalism.
3. Evidence before opinion.
4. Respect tradition while documenting diversity.
5. Every public claim must be traceable to the repository.
